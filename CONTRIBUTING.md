# Contributing

Thanks for helping improve `Paired Engineering`.

This repository is primarily a documentation and enablement project. Good contributions usually improve clarity, structure, evidence framing, portability, or the usability of the generated artifacts.

## Good contribution types

- fix broken links, typos, or formatting issues
- clarify wording that feels vague, confusing, or overly abstract
- improve examples, guides, or deck/source alignment
- improve the publication tooling or artifact generation
- propose additions that strengthen the operating model without turning the project into vendor hype or generic AI futurism

## Before making larger changes

- open an issue first if the change affects structure, naming, licensing, packaging, or the overall framework
- keep one pull request focused on one main concern when possible

## Content expectations

- prefer practical, evidence-aware improvements over hype
- critique ideas directly, but keep the tone constructive
- preserve the project's core stance: verification matters, judgment matters, and capability-building matters
- avoid adding private or identifying information from interviews, client work, or internal rollouts

## Generated artifacts

- markdown source notes are the source of truth
- generated `.pptx` files and public package outputs should be rebuilt after relevant source changes
- if you are fixing slide content, update the markdown slide source first

## Conduct

By contributing, you agree to follow the expectations in [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).
