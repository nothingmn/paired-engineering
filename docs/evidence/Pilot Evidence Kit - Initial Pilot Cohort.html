Suggested use order

  1. Read Pilot Evidence Operations Guide - Initial Pilot Cohort before kickoff.
  2. If no workplace-specific stack has been chosen yet, start from Pilot Evidence Reference Deployment - Microsoft 365 and Jira.
  3. Run Pre-Pilot Survey - Initial Pilot Cohort before kickoff.
  4. Set up Pilot Evidence Sheet - Initial Pilot Cohort with workflows, baselines, owners, cadence, and tab structure.
  5. Use Weekly Pilot Pulse - Initial Pilot Cohort during the pilot.
  6. Review a small sample of cases each week with Evidence Review Rubric - Initial Pilot Cohort.
  7. Run Post-Pilot Survey - Initial Pilot Cohort at pilot close.

What this kit should produce

By the end of a pilot, this kit should make it possible to answer:

  • which workflows improved
  • where hidden rework or review burden increased
  • where verification quality was strong or weak
  • whether support patterns are working
  • whether the next step is refine, pause, or scale

Key rule

This kit is for workflow improvement and rollout learning, not for performance management.

Default stance:

  • confidential collection where possible
  • aggregate reporting by default
  • sampled review instead of broad surveillance
