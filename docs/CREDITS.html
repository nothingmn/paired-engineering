Why this file exists

This repository uses open licensing for both content and code.

That makes reuse easier, but it also helps to state how this work should be credited in practice.

Preferred attribution for written content

When reusing the framework notes, decks, or written artifacts, prefer an attribution in this shape:

Paired Engineering. A practical operating model for AI enablement in software delivery.

If the public repository URL is available, include it.

Preferred attribution for scripts and automation

When reusing the scripts or workflow automation, preserve the license notice and credit the project as:

Paired Engineering contributors

Current attribution posture

This project currently prefers project-level attribution rather than naming an individual author in the public package.

If that changes later, update this file before the public release.
