# Security

This repository is mostly documentation plus lightweight publication tooling.

## What to report

- security issues in the scripts or workflows included here
- accidental exposure of sensitive material in generated public artifacts
- packaging mistakes that could leak private or blocked content

## What is out of scope

- disagreements with the framework or recommendations
- vulnerabilities in third-party tools that this project does not maintain
- general concerns about AI systems that are not caused by this repository's code or released artifacts

## Reporting guidance

- do not publish exploit details or sensitive exposure details immediately
- if GitHub private security reporting is enabled, use it
- if no private reporting path is available yet, open a minimal public issue asking for a private channel and do not include sensitive details

## Response intent

Security and exposure issues should be triaged quickly, especially if they affect published artifacts.
